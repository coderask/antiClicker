      - Call launcher.launch() for each in parallel (Promise.all)

    Test 2 — "cookie set in instance A is not visible in instance B":
      - Launch 2 instances
      - In instance A's context, navigate to cookie fixture, set a cookie via document.cookie
      - In instance B's context, navigate to same URL, read document.cookie
      - Assert cookie from A is NOT present in B
      - Also assert each has a distinct userDataDir path
      - closeAll() in finally

    Test 3 — "10 rapid relaunches never fail on EADDRINUSE" (serial):
      - Loop 10 times: launch() → navigate to fixture → assert geo works → close()
      - None of the 10 iterations should throw
      - Serial (no interleaving with other tests)

    Test 4 — "close event fires and cleans up profile dir":
      - Launch an instance, capture its userDataDir
      - Verify dir exists
      - Set up listener: launcher.on('instance-closed', handler)
      - Call launcher.close(id)
      - Wait for handler to fire (or check registry after close)
      - Assert userDataDir no longer exists (best-effort rm may take a moment — use polling or await close)
      - Assert launcher.list() no longer contains the instance

    Test 5 — "setGeo updates coords without relaunch":
      - Launch an instance at coords A
      - Navigate to fixture, verify coords A reported
      - Call launcher.setGeo(id, coordsB)
      - Reload the page (or navigate again), verify coords B reported
      - Instance still listed in launcher.list() with updated coords
      - close() in finally

    All tests: use headless:true override in the launcher (add a headless option to LaunchOptions,
    defaulting to false for production use, overridable in tests).

    Note: Tests 1-2 and 3 should be in separate describe blocks to keep the serial-within-itself
