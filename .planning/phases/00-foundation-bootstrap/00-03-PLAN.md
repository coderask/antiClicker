<critical_pitfalls>
- PITFALL: `0.0.0.0` exposure. The server MUST bind to '127.0.0.1', never '0.0.0.0'. Anyone on the LAN could otherwise reach the renderer (and in Phase 4 scrape the Maps API key).
- PITFALL: Hardcoded port (e.g., 3000). The server MUST `listen(0)` and read the assigned port from `server.address().port`. Tutorials default to 3000 — that collides with every dev server on the planet.
- PITFALL: Path traversal. A request for `/../../../etc/passwd` decodes to a path that escapes RENDERER_ROOT. We resolve the candidate path with `normalize` and check `candidate.startsWith(RENDERER_ROOT)`. Reject with 403 if not.
- PITFALL: Host-header attack. Even though we bind to loopback, a renderer that's somehow tricked into making a cross-origin request via a hostile DNS rebind could hit us. Reject if `req.headers.host` doesn't start with `127.0.0.1` or `localhost`.
- PITFALL: Forgetting to release the server. `app.on('before-quit')` (wired in 00-02) calls `stopRendererServer()` — but `stopRendererServer` must actually close it. Without `await server.close()`, the fd leaks across electron relaunches in test harnesses.
</critical_pitfalls>
