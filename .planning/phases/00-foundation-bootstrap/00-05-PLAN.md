<critical_pitfalls>
- PITFALL 4 from 00-RESEARCH.md: do NOT use `@electron-toolkit/preload`'s `electronAPI` pattern — it exposes raw `ipcRenderer.invoke/send/on`, defeating FND-01 invariant #4.
- PITFALL: preload runs in a sandboxed renderer context. It CANNOT use `node:fs`, `node:path`, etc. It can use `electron` (via the sandbox-safe subset: contextBridge + ipcRenderer). Test by running — if the preload throws "process is not defined" at boot, sandbox is correctly on but the preload imported something that needs Node.
- PITFALL: `process.contextIsolated` is NOT available in the renderer's main world (only in the preload's isolated world). The renderer cannot directly assert contextIsolation; the e2e in 00-06 asserts it via Playwright's `webContents.getWebPreferences()`.
- PITFALL: electron-vite emits preload as `out/preload/index.mjs` (note: `.mjs`, not `.js`) when `"type": "module"` is set in package.json. The main process loads it via `join(__dirname, '../preload/index.mjs')` (already in 00-02's index.ts). If the file name is `.js` instead, the load fails silently.
</critical_pitfalls>
