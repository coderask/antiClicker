    Create scripts/cli-prototype.ts as an ESM TypeScript file using the following structure:

    1. Imports: { parseArgs } from 'node:util', { mkdtempSync, rmSync } from 'node:fs', { tmpdir } from 'node:os', { join } from 'node:path', { chromium } from 'playwright'.

    2. parseArgs with options: lat (type: 'string'), lng (type: 'string'). If either is missing or parses to NaN, print usage to stderr and process.exit(1).

    3. Bounds validation: lat in [-90, 90], lng in [-180, 180]. Print clear error to stderr + process.exit(1) if out of range.

    4. Create userDataDir with mkdtempSync(join(tmpdir(), 'anticlicker-profile-')) so each invocation has an isolated Chromium profile.

    5. Print launch info: `Launching Chromium with geolocation lat=${lat} lng=${lng}` and `Profile dir: ${userDataDir}`.

    6. Call chromium.launchPersistentContext(userDataDir, { headless: false, geolocation: { latitude: lat, longitude: lng, accuracy: 10 }, permissions: ['geolocation'], args: ['--no-first-run', '--no-default-browser-check'] }).

    7. Open a new page via context.newPage() and navigate to 'about:blank' so there's a visible tab.

    8. Print: `Chrome launched. Ctrl-C to exit and clean up.`

    9. Cleanup function (async): sets isCleaningUp guard, calls context.close() in try/catch, calls rmSync(userDataDir, { recursive: true, force: true }) in try/catch, then process.exit(0).

    10. Register: process.on('SIGINT', cleanup) and context.on('close', () => { if (!isCleaningUp) cleanup() }).

    11. Keep the process alive with: await new Promise(() => {}) — this promise never resolves; the process stays alive until SIGINT or the browser window closes.

    The script must be runnable with: npx tsx scripts/cli-prototype.ts --lat 35.6762 --lng 139.6503
  </action>
