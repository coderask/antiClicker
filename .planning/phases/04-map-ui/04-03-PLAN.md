    Export:
      interface ParsedCoords { latitude: number; longitude: number }
      export function parseMapsUrl(url: string): ParsedCoords | null

    Logic:
    1. Reject if not a google.com/maps or maps.google.com URL (case-insensitive).
       Check: url.includes('google.com/maps') || url.includes('maps.google.com')
    2. Try @lat,lng pattern in path (covers all /maps/@..., /maps/place/.../@... forms):
       const atMatch = url.match(/@(-?\d+\.?\d*),(-?\d+\.?\d*)/)
    3. If no atMatch, try query param q= or ll=:
       const qMatch = url.match(/[?&](?:q|ll)=(-?\d+\.?\d*),(-?\d+\.?\d*)/)
    4. Parse floats from matched groups.
    5. Validate: -90 ≤ lat ≤ 90, -180 ≤ lng ≤ 180. Return null if out of range.
    6. Return { latitude, longitude } or null.

    No external libraries. No try/catch needed (regex won't throw on bad input).
  </action>
  <done>parseMapsUrl.ts exports the function; handles all 6 URL patterns from the spec.</done>
