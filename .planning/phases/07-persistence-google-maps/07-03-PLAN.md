### Persist recentPins with 500ms debounce on change

In `src/renderer/src/App.tsx`:
- Add a new useEffect that watches `recentPins` and debounces the electron-store write:
  ```ts
  useEffect(() => {
    // Debounce persist — only write after 500ms of no further changes
    // (avoids electron-store IO on every pin drop)
    if (recentPins.length === 0) return; // Skip initial empty array (loaded from store already)
    const timer = setTimeout(() => {
      // Add timestamp to each pin for the storage schema
      const toStore = recentPins.map((p) => ({
        latitude: p.latitude,
        longitude: p.longitude,
        timestamp: Date.now(),
      }));
      window.api.setRecentPins(toStore).catch(() => undefined);
    }, 500);
    return () => clearTimeout(timer);
  }, [recentPins]);
  ```

Note: The existing `pushBounded` call in `handleLaunch` already handles the ring buffer logic (cap at 10). The debounced persist just writes whatever the current `recentPins` state is.
</task>
