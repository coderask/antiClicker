### Generate app icon

Create `scripts/gen-icon.mjs` — a Node.js script that generates a 512x512 PNG using raw pixel manipulation (no canvas library needed for a solid-color placeholder). The PNG format header + IDAT chunks can be written manually, or better: use the `node:zlib` + manual PNG encoder approach for a solid color.

Actually, simplest approach: write a minimal PPM to PNG conversion, or just commit a pre-baked base64-encoded small PNG. For maximum reliability, we'll write the PNG using raw bytes encoded in the script.

The icon should be:
- 512x512 pixels
- Background: #2d5a8e (AntiClicker blue)
- Simple solid color (no text — keep it reliable without canvas)

Since generating a binary PNG programmatically from scratch in a script is complex without `canvas`, the simplest approach is to create a minimal valid PNG via a buffer. We'll use a 1x1 PNG scaled up, or better yet just write a small but valid PNG.

**Alternative**: Use the `sharp` package OR just commit a pre-baked PNG as base64 in the gen-icon.mjs and decode it.

Best approach: embed a pre-baked minimal 512x512 solid-color PNG as a base64 string in the script and write it to disk. This is fully self-contained.

Create `scripts/gen-icon.mjs` that:
1. Contains a base64-encoded 16x16 solid blue PNG (or generates one via PNG raw bytes)
2. Writes it to `build/icon.png`

Then run `node scripts/gen-icon.mjs` to produce the icon.

Actually, the cleanest approach is to generate a proper 512x512 PNG using raw bytes + zlib compression via Node's built-in `zlib.deflateSync`. This works without any npm dependency.
